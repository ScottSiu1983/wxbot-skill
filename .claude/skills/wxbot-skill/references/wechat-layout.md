# WeChat 桌面 UI 布局参考

## 屏幕信息

- **逻辑分辨率**: 1470 × 956（pyautogui 坐标空间）
- **物理分辨率**: 2940 × 1912（2x Retina，截图原始尺寸）
- **坐标系**: pyautogui 使用逻辑坐标，local_vision.py 已自动转换

## WeChat 窗口

窗口位置通过 AppleScript 动态获取，不硬编码：

```applescript
tell application "System Events"
    tell process "WeChat"
        set w to window "微信"
        set p to position of w  -- (x, y)
        set s to size of w      -- (w, h)
    end tell
end tell
```

典型值（会随用户调整而变化）：`x≈346, y≈33, w≈1124, h≈923`

## WeChat 内部布局（相对窗口左上角偏移）

```
┌──────────────────────────────────────────────┐
│  图标侧栏  │  聊天列表面板  │   内容区域       │
│  0~60px   │  60~280px     │  280px~末尾      │
│           │               │                  │
│  微信图标  │  DuckDB群      │  消息内容         │
│  通讯录   │  Kent          │  [对方左对齐]     │
│  发现     │  工作群        │  [我方右对齐]     │
│  我       │  ...           │                  │
│           │               │  [输入框]         │
└──────────────────────────────────────────────┘
```

## 各区域坐标计算（以窗口矩形 wx,wy,ww,wh 为基准）

| 区域 | X 起点 | Y 起点 | 宽度 | 高度 |
|------|--------|--------|------|------|
| 聊天列表 | wx+60 | wy+60 | 220 | wh-80 |
| 内容区域 | wx+280 | wy+40 | ww-280 | wh-80 |
| 内容标题 | wx+280 | wy+30 | ww-280 | 50 |
| 输入框区域 | wx+280 | wy+wh-80 | ww-280 | 80 |

## 消息方向判断

- **对方发的消息**（左对齐）: `item.x < 内容区中线`
- **我发的消息**（右对齐）: `item.x >= 内容区中线`
- 内容区中线 = `wx + 280 + (ww - 280) / 2`

## 输入框定位

首选：OCR 查找"发送"按钮，点击其左侧约 100px 处
备用：点击 `(内容区中央 x, wy + wh - 45)`

## 已知 UI 噪音（OCR 过滤规则）

以下内容在消息解析时应忽略：
- 时间戳：`^\d{1,2}:\d{2}$`（如 `14:30`）
- 日期分隔：`昨天`、`今天`、`星期X`
- 按钮文字：`发送`、`Search`
- 系统格式：`[链接]`、`[图片]`（由系统生成）
- 分隔线：`---`

## WeChat Cmd+F 搜索行为

1. `Cmd+F` 打开搜索框（顶部）
2. 输入名称后约 0.5s 显示结果
3. 结果分组：联系人（顶部）、聊天记录（下方）
4. 点击联系人结果 → 打开对话
5. `Escape` 关闭搜索

## 注意事项

- WeChat 窗口可能被其他窗口遮挡 → 每次操作前先 `activate`
- 新消息到来会使聊天列表滚动 → 导航后必须验证标题
- 搜索框的"Kent"自动补全 → 按 Enter 而非依赖下拉选择
- 图片/表情在 OCR 中不可见 → 标记为 `[图片]` 或直接忽略
