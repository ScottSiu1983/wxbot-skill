## 2. 窗口布局分析

### 2.1 WeChat 窗口结构

```
┌────────────────────────────────────────────────────────────────┐
│                        标题栏 (y: 0~40)                        │
├──────┬─────────────────────┬────────────────────────────────────┤
│      │                     │                                    │
│ 侧   │     聊天列表         │           内容区                   │
│ 边   │   (chatlist)        │         (content)                  │
│ 栏   │                     │                                    │
│      │   x: 60~(动态)      │   x: (动态)~ww                     │
│ 60px │   y: 60~wh-80       │   y: 40~wh-80                      │
│      │   宽度用户可调       │   (33%~75% 窗口宽度)               │
│      │                     │                                    │
│      │  ┌─────────────┐   │   ┌─────────────────────────────┐  │
│      │  │ 搜索栏      │   │   │                             │  │
│      │  └─────────────┘   │   │  对方消息 (them, x < mid)   │  │
│      │  ┌─────────────┐   │   │                             │  │
│      │  │ 聊天项 1    │   │   │        ───────────          │  │
│      │  ├─────────────┤   │   │                             │  │
│      │  │ 聊天项 2    │   │   │  我的消息 (me, x >= mid)    │  │
│      │  ├─────────────┤   │   │                             │  │
│      │  │ ...         │   │   │                             │  │
│      │  └─────────────┘   │   └─────────────────────────────┘  │
├──────┴─────────────────────┴────────────────────────────────────┤
│                    输入框区域 (y: wh-80~wh)                     │
│              点击位置: ww-25, wh-25 (窗口右下角)                │
└────────────────────────────────────────────────────────────────┘
```

### 2.2 关键坐标常量

```python
SIDEBAR_W   = 60    # 侧边栏宽度（固定）
SETTLE      = 0.35  # 点击后等待 UI 稳定的秒数
MAX_RETRIES = 3     # 导航最大重试次数
```

### 2.3 动态坐标计算

| 函数 | 计算 | 说明 |
|------|------|------|
| `_detect_chatlist_width()` | 像素颜色检测 | 扫描背景色差异，精确定位右边界（~50-100ms） |
| `_chatlist_x_range()` | `(wx + 60, wx + 60 + chatlist_w)` | 聊天列表的 x 范围 |
| `_content_x_min()` | `chatlist_x_range[1]` | 内容区左边界 = 聊天列表右边界 |

### 2.4 功能区域边界

| 区域 | x 范围 | y 范围 |
|------|--------|--------|
| 标题栏 | `wx + SIDEBAR_W` ~ `wx + ww` | `wy` ~ `wy + 40` |
| 聊天列表 | `wx + SIDEBAR_W` ~ `chatlist_right` | `wy + 60` ~ `wy + wh - 80` |
| 内容区 | `content_x_min` ~ `wx + ww` | `wy + 40` ~ `wy + wh - 80` |
| 输入框 | `content_x_min` ~ `wx + ww` | `wy + wh - 80` ~ `wy + wh` |

## 3. 命令流程详解

### 3.1 `chat list` — 列出可见聊天

```
┌─────────────────┐
│  chat list 开始 │
└────────┬────────┘
         ▼
┌─────────────────┐
│ 激活 WeChat     │ ← AppleScript: tell application "WeChat" to activate
└────────┬────────┘
         ▼
┌─────────────────┐
│ 获取窗口 rect   │ ← AppleScript: position + size of window "微信"
└────────┬────────┘
         ▼
┌─────────────────┐
│ 全屏 OCR        │ ← get_screen_text(mode="accurate")
└────────┬────────┘
         ▼
┌─────────────────┐
│ 筛选聊天列表区域 │ ← x: list_x_min~list_x_max, y: wy+60~wy+wh-80
└────────┬────────┘
         ▼
┌─────────────────┐
│ 过滤噪音        │ ← 时间戳、日期、"发送"、"搜索"、纯数字等
└────────┬────────┘
         ▼
┌─────────────────┐
│ 返回聊天名称    │ → [OK] 8 chats: Kent | 工作群 | 妈妈 | ...
└─────────────────┘
```

**噪音过滤规则** (`_NOISE_PATTERNS`):
```python
r"^\d{1,2}:\d{2}$"           # 时间戳 HH:MM
r"^\d{1,2}月\d{1,2}日$"       # 日期
r"^(昨天|今天|星期...)$"      # 相对日期
r"^发送$"                     # UI 按钮
r"^搜索$"                     # 搜索栏标签
r"^\[.*?\]$"                  # 系统通知如 [链接]
r"^\d+$"                      # 未读消息角标
```

### 3.2 `chat read` — 读取聊天内容

#### 3.2.1 主流程

```
┌─────────────────┐
│  chat read 开始 │
└────────┬────────┘
         ▼
┌─────────────────────┐
│ _navigate_to_chat() │ ← 导航到目标聊天
└────────┬────────────┘
         │ 成功
         ▼
┌─────────────────────┐
│ 获取窗口 rect       │
└────────┬────────────┘
         ▼
┌─────────────────────┐
│ _is_group_chat()?   │ ← 检测是否为群聊（标题栏是否有 (N) 成员数）
└────────┬────────────┘
         │
    ┌────┴────┐
    ▼         ▼
 群聊       1:1 聊天
    │         │
    ▼         ▼
┌────────┐  ┌────────────────┐
│两步读取│  │ 单步读取        │
│+ 合并  │  │ (消息不足时滚动)│
└────┬───┘  └───────┬────────┘
     │              │
     └──────┬───────┘
            ▼
┌─────────────────────┐
│ _parse_messages()   │ ← 解析消息 + 检测视觉元素
└────────┬────────────┘
         ▼
┌─────────────────────┐
│ _summarize()        │ ← 生成极简摘要
└────────┬────────────┘
         ▼
      返回结果
```

#### 3.2.2 群聊 vs 1:1 读取差异

| 特性 | 群聊 | 1:1 聊天 |
|------|------|----------|
| 读取策略 | 两步：底部最新 + 向上滚动历史 | 单步，不足 5 条时向上滚动 |
| 滚动页数 | 3 页 | 1~2 页 |
| 发送人识别 | 昵称（x 偏左 + 字体高度小） | x 位置（midline 左/右） |
| 消息去重 | 文本内容去重 | 文本内容去重（滚动后合并） |

### 4.2 重试机制

| 阶段 | 重试条件 | 最大次数 |
|------|----------|----------|
| 整体导航 | 点击/搜索后验证失败 | 3 次 |
| 单次验证 | 首次验证失败后等 0.5s 再验证一次 | 2 次 |
| OCR 模式 | fast 模式失败后用 accurate 模式 | 2 次 |

### 4.3 搜索栏点击策略

```python
def _click_search_bar(rect):
    # 优先：OCR 查找 "搜索" 文字（fast 模式 ~300ms）
    candidates = [OCR 结果在聊天列表顶部 60px 内]

    if candidates:
        点击 OCR 识别到的坐标
    else:
        # Fallback：硬编码位置
        x = (list_x_min + list_x_max) // 2
        y = wy + 30
        点击 (x, y)
```

## 5. 验证流程 (`_verify_chat_open`)

### 5.1 验证原理

通过 OCR 读取标题栏文字，检查是否包含目标名字或其子串。

```
┌────────────────────────────────────────────────────┐
│                    标题栏                          │
│  x: wx+SIDEBAR_W ~ wx+ww, y: wy ~ wy+40           │
│                                                    │
│  ┌──────────────────────────────────────────────┐ │
│  │  情怀新群 (12)              ⋯  □  ×         │ │
│  └──────────────────────────────────────────────┘ │
└────────────────────────────────────────────────────┘
```

**重要修复**：标题栏 x 范围从 `wx + SIDEBAR_W` 开始（侧栏右边界），而不是 `_content_x_min(rect)`（聊天列表右边界），确保能正确匹配标题栏文字。

### 5.2 关键词生成策略

```python
keywords = [
    name,                    # 完整名字 "情怀新群"
    name[:len-1],           # 逐步缩短 "情怀新"
    name[:len-2],           # "情怀"
    ...,
    词组拆分,                # "情怀", "新群"（通过正则提取）
]
```

### 5.3 OCR 模式选择

| 模式 | 耗时 | 使用场景 |
|------|------|----------|
| fast | ~300ms | 快速检查（_verify_chat_open 参数控制） |
| accurate | ~1000ms | 精确验证（默认） |

## 6. 消息解析流程 (`_parse_messages`)

### 6.1 发送人识别

**1:1 聊天**:
```python
midline_x = content_x + content_w // 2
sender = "me" if item["x"] >= midline_x else "them"
```

**群聊**:
```python
# 基于视觉特征区分昵称和消息
# 昵称：x 偏左（靠近头像）+ 字体高度小
is_nickname = (
    x < nickname_x_max      # 靠近头像（content_x + 140）
    and h < nickname_h_max  # 小号字体（中位数 * 0.9）
    and len(text) <= 20     # 昵称不会太长
    and not re.search(r'[，！？；：、]', text)  # 不含句内标点
)
```

### 6.2 视觉元素检测

```
┌─────────────────────────────────────────────┐
│          _detect_visual_elements()          │
├─────────────────────────────────────────────┤
│  1. 截图内容区域                             │
│  2. 计算背景色（四角中位色）                  │
│  3. 色差 > 40 的像素 = 非背景内容            │
│  4. 排除 OCR 文字区域                        │
│  5. 连通域分析找像素簇                       │
│  6. 排除头像（正方形 + 靠近边缘）            │
│  7. 排除消息气泡（低色彩方差 < 20）          │
│  8. 分类：                                   │
│     - w>80 & h>60 → [图片] + VNClassify     │
│     - w>15 & h>15 → [表情]                  │
└─────────────────────────────────────────────┘
```

### 6.3 消息合并

```python
# 相邻同 sender 且 Y 差距 < 25px 的条目合并
if (messages[-1]["sender"] == item["sender"]
    and abs(item["_y"] - messages[-1]["_y"]) < 25):
    messages[-1]["text"] += " " + text
```

## 7. OCR 模式详解

### 7.1 模式对比

| 特性 | fast | accurate |
|------|------|----------|
| Vision 框架级别 | VNRequestTextRecognitionLevelFast (1) | VNRequestTextRecognitionLevelAccurate (0) |
| 典型耗时 | 100-300ms | 800-1000ms |
| 中文识别质量 | 一般（可能有乱码） | 高 |
| 适用场景 | 快速查找、初步验证 | 精确匹配、消息解析 |

### 7.2 使用分布

| 函数 | fast | accurate |
|------|------|----------|
| `_click_search_bar` | ✓ | |
| `_navigate_to_chat` (Step 1) | ✓ | |
| `_navigate_to_chat` (Step 2) | | ✓ |
| `_verify_chat_open` | ✓ (可选) | ✓ |
| `_is_group_chat` | ✓ | |
| `_read_content_area` | | ✓ |
| `chat_list` | | ✓ |

**注意**：搜索结果面板的中文在 `fast` 模式下乱码严重，必须用 `accurate`。

### 7.3 滚动机制

```
滚动方式: CGEvent 像素级平滑滚动（模拟真实触摸板）
- 不使用 pyautogui.scroll()（基于滚轮事件，容易被检测）
- 使用 CGEventCreateScrollWheelEvent + kCGScrollEventUnitPixel
- 分解为 5-10 个小事件，添加随机抖动
- 1 页 = 内容区高度 - 60px（屏幕高度相关）
```

## 8. 调试系统

### 8.1 Debug 目录结构

同一轮 Claude 对话的多次 skill 调用共享同一个会话目录，避免每次调用都创建新目录。每轮对话结束后自动清理旧会话。

```
.claude/skills/wxbot-skill/debug/
├── .session                      # 当前会话跟踪文件
├── .session_{session_id}         # 各会话的元数据文件
├── 20260331_143052/              # 会话目录（按时间戳命名）
│   ├── 143052_chat_read/         # 一次 chat read 调用
│   │   ├── log.txt               # 时序日志
│   │   ├── 01_nav_attempt1_before.png
│   │   └── ...
│   └── 143120_chat_reply/        # 一次 chat reply 调用（同会话）
│       └── ...
└── 20260331_150000/              # 下一轮 Claude 对话的会话
    └── ...
```

**会话超时**：超过 1 小时无活动视为新会话。

### 8.2 日志格式

```
[14:30:52.123] T=0.00s +0.00s | === WeChat Skill Debug ===
[14:30:52.123] T=0.00s +0.00s | Session: 20260331_143052
[14:30:52.123] T=0.00s +0.00s | Command: chat_read
[14:30:52.123] T=0.00s +0.00s | Args: {"name": "Kent"}
[14:30:52.456] T=0.33s +0.33s | Window rect: (735, 33, 735, 923)
[14:30:52.789] T=0.67s +0.33s | Screenshot: 01_nav_attempt1_before.png
[14:30:53.012] T=0.89s +0.22s | OCR [find_Kent]: 2 items
[14:30:53.012] T=0.89s +0.00s |   x= 180 y=150 h=14 c=0.95 | Kent
```

- `Session:` : 所属会话 ID
- `T=` : 从命令开始的总耗时
- `+` : 距上一条日志的增量耗时

### 8.3 环境变量

```bash
WECHAT_DEBUG=1              # 默认开启
WECHAT_DEBUG=0              # 关闭调试
WECHAT_DEBUG_MAX_ROUNDS=5   # 保留最近 N 轮会话（默认10）
```

超过 `WECHAT_DEBUG_MAX_ROUNDS` 的旧会话目录会被自动清理。

### 9.1 锁文件

### 9.2 锁定逻辑

```python
# chat read/reply 需要互斥执行
lock_fd = open(LOCK_FILE, "w")
try:
    fcntl.flock(lock_fd, fcntl.LOCK_EX | fcntl.LOCK_NB)
except BlockingIOError:
    print("[ERR] 另一个 wechat.py 实例正在操作中")
    sys.exit(1)
```

### 9.3 为什么需要锁

- pyautogui 控制鼠标键盘，并发执行会导致混乱
- OCR 截图时窗口状态必须稳定
- 避免多个 Claude 会话同时操作微信

### 15.2 标题栏验证区域错误

**问题**：`_verify_chat_open` 使用 `_content_x_min(rect)` 作为标题栏 x 起点，这实际上是聊天列表右边界，导致标题栏文字（在侧栏右侧）无法被正确匹配。

**修复**：改用 `wx + SIDEBAR_W` 作为标题栏 x 起点。

```python
# 改前
cx_min = self._content_x_min(rect)  # 聊天列表右边界

# 改后
cx_min = wx + SIDEBAR_W  # 侧栏右边界
```

## 16. chatlist 宽度检测策略

### 16.1 问题背景

用户可手动调整 WeChat 窗口中 chatlist 和 content 区域的宽度比例，范围约 33%~75%。如果使用固定比例估算，可能导致：
- chatlist 区域 OCR 结果被误认为 content 区域
- content 区域左侧消息被误过滤

### 16.2 检测方案：像素颜色扫描

```
原理: WeChat 的 chatlist 和 content 区域背景色不同
     扫描 y 中线，找到背景色变化的 x 坐标

步骤:
  1. 截图（~50ms）
  2. 取 chatlist 典型背景色（采样点）
  3. 从最小宽度位置开始向右扫描
  4. 找到颜色差异 > 30 的点即为边界
  5. 计算 chatlist_w = 边界x - wx - SIDEBAR_W

延迟: ~50-100ms
fallback: 如果扫描失败，使用 33% 估算
```

### 16.3 为什么不用 OCR

- OCR 检测需要 300-500ms
- chatlist 宽度用于多处的 x 范围过滤
- 像素扫描延迟低、准确度高

## 17. AI 交互与回复哲学

为了确保「AI 分身」的回复具有真实的人感（人味）并符合即时通讯（IM）的特征，系统遵循以下交互原则：

### 17.1 极简主义 (Minimalism)

- **字数控制**：严禁生成长篇大论、总结式或汇报式的长难句。
- **单点突破**：一条消息只说一件事，或者只针对当前上下文中最具体、最新的一个观点进行回复。

### 17.2 拟人化节奏 (Human-like Rhythm)

- **你来我往**：模拟“你一句我一句”的自然对话流，而不是一次性给出完整答案。
- **口语化**：使用自然口语（如“真绝了”、“带感”、“顺应潮流”），避免过于机械和礼貌的 AI 腔调。

### 17.3 身份锚定 (Identity Alignment)

- **分身自觉**：回复内容应与其 `[AI分身]` 的身份前缀相符。例如，在讨论 AI、自动化或 Avatar 相关话题时，利用身份关联性增加互动质量。
- **情境感**：根据群聊或私聊的氛围调整语气，但在任何情况下都优先保持简洁。

### 17.4 输入仿真

- **词组分块**：如 `computer_use.py` 中实现的，模拟人脑思考词组的过程，分块进行 Unicode 键入。
- **呼吸停顿**：在标点符号后增加较长停顿，模拟真实打字中的顿挫感。
